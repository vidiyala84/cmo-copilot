# CMO Copilot — Implementation Summary

**What it is:** a marketing-budget decision agent that turns a fuzzy CMO question
("ROAS moved — what do I do?") into a reliable, auditable recommendation over a
realistic account (~300 campaigns, $1.2M/mo), built on Qwen (Alibaba Cloud Model
Studio).

---

## Implementation

### Measurement substrate (built first, deliberately)

- **Deterministic synthetic account:** 300 campaigns rolled into 5 groups, 90 days,
  `seed=42` (`cmo/config.py`, `cmo/datagen.py`). Group rollup keeps model context
  **O(groups), not O(campaigns)** — dumping the raw account blows the 32k window at
  ~500–1k campaigns.
- **100-question self-validating benchmark** (`cmo/benchmark.py`): 40 simple / 40
  medium (traps) / 20 complex (multi-item plans), **generated by Python with no model
  in the generation loop**. Every "correct" answer is **probe-verified present in the
  data before it counts**. Scoring is 0.4 root-cause + 0.4 action + 0.2 sourcing;
  multi-item plans scored by **F1** against the reference plan.

### Engine

- **Qwen via an OpenAI-compatible tool-calling loop** (`cmo/llm.py`): `qwen-plus`
  orchestrate, `qwen-flash` cheap sub-tasks, `qwen-max` synthesis, `text-embedding-v3`
  for memory retrieval.
- **Tool belt** (`cmo/tools.py`, 7 tools): `get_campaign_metrics`, `diagnose_drivers`,
  `find_opportunities`, `recommend_portfolio`, `forecast_impact`, `propose_reallocation`
  (**constraint-checked** — brand floor, max weekly shift %, learning-phase guard),
  exposed over an **MCP server** (`cmo/mcp_server.py`).

### The architecture comparison (the experiment)

Seven approaches, identical 100 questions, live on Qwen:

| Approach | Overall |
|---|---|
| direct — just ask Qwen | 27% |
| + rules in the prompt | 51% |
| single agent + tools | 57% |
| **6-agent society** (Analyst / Forecaster / Risk-Officer-with-veto / Portfolio-Planner / Growth-Lead / Coordinator) | 66% |
| LLM planner | 71% |
| tool-derived (structured) | 76% |
| **composed gated planner** | 100% |

The finding: **no single architecture wins all tiers — the failure modes are
opposite.** Rules + the veto crack the *traps* (when to *hold*); tools + planning crack
the *fixes/plans* (what to *do*). Composing them solves everything — but that
**hand-codes** the gates.

### The centerpiece — a memory that *learns* the gates (`tracks/track1/memory_gates.py`)

- **`observe()`** logs raw features per episode (funnel drops, elasticity, ROAS spread,
  flags, opportunity lift) + the outcome.
- **`fit_gates()`** trains a **scikit-learn `DecisionTreeClassifier` on the accumulated
  outcome history each session** — the tree *discovers the thresholds itself* (e.g.
  `cvr_drop > 0.53 → fix_tracking`; `0.15–0.53 → fix_targeting`). No hand-set numbers.
- A leaf becomes an enforced gate **only if history shows it *mattered*** (validated by
  support / purity / **lift > 0** — the base was really getting those cases wrong).
- Enforcement is a **deterministic feature match — 0 LLM calls.** Composed with the
  society: **74% → 100% over four sessions, and it holds.**

### Deployment

FastAPI backend on **Alibaba Cloud ECS (Docker)** serving an interactive demo (pick a
scenario → single-model-vs-society run → steps + verdict), the JSON API, and the MCP
server. **170 offline tests**, fully reproducible with no credentials.

## What's novel

1. **Learned, validated, deterministically-enforced override gates — not reflection
   memory.** The standard "agent memory" is prose the model writes itself and re-reads
   ("last time X, so do Y"). We show that **discovers the right rules but oscillates and
   degrades** (30→44→46→32→31% across sessions — reported as an **honest negative
   result**). The working memory instead learns each gate as a **structured condition
   from a decision tree over its own episode outcomes**, keeps only the gates its history
   proves changed the result, and enforces them as a **deterministic match**. It is
   memory that learns *which of its own calls to override* — precise, auditable, and it
   converges where prose never does.

2. **The memory overrides a multi-agent society, not a single model.** The learning
   layer sits *on top of* a specialist debate — it closes the society's residual blind
   spot rather than trying to be the whole reasoner.

3. **Base-agnostic learning.** The tree learned on a **live-Qwen base is identical** to
   the one learned on a deterministic base — because it learns from the **fixed
   ground-truth outcomes, not the noisy model** (it learns the *task's* structure).
   Consequence: it's stable regardless of the engine underneath, and **enforcement gets
   cheaper as it learns** (learned decisions cost zero calls).

4. **Composition-as-a-finding, then learned instead of hand-coded.** The result that *no
   single architecture wins all tiers* (traps and plans fail oppositely) is itself a
   contribution; hand-composing a gated planner reaches 100% but hard-codes the traps —
   which is exactly the **overfitting** the learned memory is designed to avoid (a real
   business's traps aren't known in advance).

5. **Rigorous, honest measurement.** A benchmark generated deterministically with **no
   model in the generation** and **every answer proven in the data before it scores**,
   plus **negative results reported in full** rather than hidden — the mechanism (clean
   climb) and the noisier live numbers kept separate, never substituted.

---

> **In one line:** reliability didn't come from a smarter model — it came from a
> specialist society that reasons plus a memory that learns, from its own outcomes,
> which of its calls to override, validated against history and enforced for free.
